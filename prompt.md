# 话术生成网页样式指南

## 1. 总体风格

- **目标**: 打造一个专业、简洁、高效的SaaS产品界面。
- **原则**: 遵循极简主义设计，移除所有不必要的装饰性元素，聚焦于核心功能和内容。
- **布局**: 采用清晰、有组织的网格布局，确保页面结构的稳定性和可预测性。留出足够的空白（留白）以增强可读性，避免信息过载。

## 2. 色彩方案

- **主色调**: 使用中性色作为基础，如灰色、白色和深灰色。
- **强调色**: 选择一个饱和度较低、不刺眼的单一颜色作为品牌色或操作提示色（例如，蓝色或绿色），用于按钮、链接和重要状态指示。
- **禁用颜色**: 严禁使用高饱和度、花哨的颜色，如紫色、粉色、橙色等，以保持界面的专业性。
- **背景色**: 主要使用白色或非常浅的灰色（如 `#f4f5f7`），为内容提供一个干净的背景。

## 3. 字体与排版

- **字体**: 选择一款现代、清晰的无衬线字体（如 Inter, Helvetica, Arial）。
- **字号**: 建立明确的字号层级，例如：
    - 页面标题: 24px
    - 卡片/区域标题: 18px
    - 正文文本: 14px
    - 辅助/注释文本: 12px
- **行高**: 设置 `1.5` 到 `1.7` 的行高，保证大段文本的可读性。

## 4. 组件风格

- **按钮**:
    - 主要按钮: 实心填充，使用强调色。
    - 次要按钮: 线框样式或浅灰色填充。
    - 状态: 明确区分 `hover`, `focus`, `disabled` 状态。
- **输入框**:
    - 边框: 使用浅灰色边框。
    - 焦点: `focus` 状态时，边框颜色变为强调色。
    - 统一性: 所有输入框、下拉选择器、文本域等表单组件样式保持高度统一。
- **卡片/容器**:
    - 使用微妙的阴影（不要过重）或细微的边框来区分层级。
    - 圆角: 使用统一的小圆角（如 4px 或 6px）为界面带来柔和感。

## 5. 统一性

- **全局应用**: 以上所有规则必须在整个应用的每一个页面和组件中得到统一执行。
- **组件库**: 建议基于此指南建立一套可复用的UI组件库，以确保未来开发的一致性。
